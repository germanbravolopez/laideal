Releases:

9.3
- Improvement: a partial payment ("Cobrar") left unconfirmed because the AEAT was slow or unreachable is now recovered at startup like any other invoice
  - Previously the startup recovery list only offered invoices submitted when saving the ticket; a timed-out partial payment had to be reconciled by hand at the AEAT site. The app now lists each pending payment event with its own invoice number (e.g. 3245-1) so you can retry, mark as error, or postpone it.
- Improvement: the annual accounting (Contabilidad) report is generated faster
  - The yearly report now reads each table in a single grouped pass instead of scanning it once per quarter. The totals, IVA and counts are exactly the same.

9.2
- Bug fix: charging a partial payment ("Cobrar") that is the first payment of a ticket now sends the AEAT the same invoice number it stores, so reprinting that invoice's QR or cancelling it works correctly
- Change: "Cobrar" now prints a single invoice (the customer copy) instead of two - use the print button in Recogida de prendas if you need another copy
- Improvement: euro amounts in the accounting and listado PDFs now show the thousands separator (e.g. 1.234,56 €)

9.1
- Bug fix: reprinted invoices for partial or late payments now show a QR with the correct date
  - The reprinted factura's QR used the reception date instead of the payment date, so for a payment made on a different day than the garment was received the QR did not match the invoice registered at the AEAT. It now uses the payment date, matching what was sent.
- Bug fix: a payment is no longer marked as failed when the AEAT is slow to reply
  - If the AEAT did not answer within a few seconds while charging through "Cobrar", the invoice was recorded as an error. It is now kept as pending (the submission may still have arrived), so it is not wrongly shown as failed and can be reconciled later.
- Bug fix: closing a quarter's accounting reliably locks it
  - When a quarter had income only in its first months and nothing in the last month, "Contabilidad" treated the quarter as empty and did not lock it. It now checks the whole quarter.
- Improvement: the app no longer freezes briefly during the automatic backup at startup
  - The daily backup now runs in the background, so the window stays responsive while it is created.

9.0
- New: partial payment of a ticket ("Cobrar" in Recogida de prendas)
  - "Pagar todo" now opens a Cobrar dialog listing every unpaid garment of the ticket, each pre-selected. Untick the garments the client is not paying this time and press Cobrar: only those are charged and a Verifactu invoice is issued for that payment. The rest stay unpaid and can be charged later as a separate payment, each with its own invoice number (e.g. 3245-1, 3245-2). The printed factura covers exactly the garments paid in that event.
- New: cancel a specific Verifactu invoice of a ticket (Herramientas -> Anular factura Verifactu)
  - A ticket paid in several events now shows one line per payment, each with its own Anular button, so you can cancel a single event with the AEAT instead of the whole ticket.
- New: pending Verifactu submissions are recovered at startup
  - If a ticket was saved while Verifactu was unreachable or not configured, the app now lists those tickets on startup and lets you retry, mark as error, or postpone each one. Can be limited by date and switched off in Configuracion.
- New: automatic database backups
  - The app makes a verified backup of the database on startup (kept daily for 30 days, monthly for 4 years), and you can make one on demand from Herramientas -> Hacer copia de seguridad ahora. Backups are stored next to the database.
- New: landline phone field in the Recogida de prendas form.
- Improvement: clearer accounting (Contabilidad) reports
  - The quarterly / monthly / annual PDF now shows the IVA settlement (IVA repercutido vs soportado), the net result of the period, the date range, and the number of tickets and invoices, plus a consolidated annual summary. The listados PDFs were restyled to match.
- Improvement: the Verifactu ServiceKey is now stored encrypted
  - The AEAT ServiceKey in the settings file is encrypted for the current Windows user instead of kept as plain text; an existing key is migrated automatically. Note: after reinstalling Windows or moving to another user/PC the key must be entered again.
- Bug fix: "Recoger todo" and "Pagar todo" no longer affect other clients' tickets when searching by name
  - When a name search returned several clients' tickets, these buttons could act on every loaded ticket. They now act only on the selected ticket.
- Bug fix: size-dependent garments (e.g. cortinas charged per m2) are priced correctly with comma decimals
  - A size like "2,6" entered with a comma is no longer read as 0, so the charged amount matches the stored size.
- Bug fix: the garment claim receipt (recibo) no longer shows a Verifactu QR
  - The QR belongs on the factura (tax document), not on the claim ticket.

8.4
- Bug fix: when searching by client name in Recogida de prendas, marking a row as entregada or splitting a garment no longer affects the wrong ticket
  - The table's column headers are clickable to sort, and the default sort is by ticket number descending. Previously, after any sort changed the visible order, clicking a row stored the visible position instead of the underlying ticket, so the next action (Entregar, Separar prenda) was applied to a different ticket than the one shown. Now the underlying ticket is always resolved before any database write. Pagar todo and Recoger todo were already correct because they iterate every loaded row.
- Improvement: column-header sort in Recogida de prendas now orders dates chronologically and Importe numerically
  - Date columns previously sorted as text (so "31-01-2026" came before "02-02-2026") and Importe sorted as text too (so "100" came before "9"). Sorting now uses the actual date and numeric value.

8.3
- New: in-app updater
  - Ayuda -> Buscar actualizaciones... checks GitHub for a newer release and, if you confirm, downloads the installer and runs it. The app closes and the installer replaces it in place - no manual download or uninstall step. Your database, settings (~/.laideal_settings.json) and log (~/.laideal.log) are preserved.
  - The same check also runs silently at startup so a new release is surfaced as soon as it is available. Turn it off in Archivo -> Configuracion -> "Buscar actualizaciones al iniciar la aplicacion" if you prefer manual checks.
- New: Ayuda -> Notas de la version... shows the full history of changes shipped in every release, offline. Same content shown by the installer at install time.

8.2
- Bug fix: factura is no longer auto-printed when pressing "Pagar todo"
  - 8.1 added an automatic print right after "Pagar todo" plus a "second copy?" prompt, but Verifactu's QR has to come back from the AEAT relay first; if the network was slow or the endpoint blipped, the factura sometimes printed without a QR. The print button on the RecogPrendas window stays - press it when the row shows ENVIADA so the QR is on the receipt. The second-copy prompt moves with the auto-print and is gone too; click the print button twice for the second copy.

8.1
- Bug fix: search by client name in Recogida de prendas now returns all the tickets for that client
  - Previously, for clients with many receipts some tickets could be silently hidden from the result; the search is now complete regardless of how many rows the ingresos table has.
- New: "Pagar todo" now automatically prints the factura and asks if a second copy is needed
  - After all garments are marked paid, the factura is sent to the printer and a dialog "¿Imprimir una segunda copia de la factura?" offers the establishment copy
  - The print button no longer closes Recogida de prendas; it resets the form instead, so chaining to the next client does not require reopening the window
- Build / release tooling (internal): single PowerShell command (releases\release.ps1 X.Y) now builds, runs windeployqt, zips and produces the Inno Setup installer in one step; build switched from MinGW Makefiles to Ninja to avoid antivirus blocks on %TEMP%\*.bat; README Build and Release procedure sections rewritten to match

8.0
- New feature: Verifactu integration - AEAT mandatory digital invoicing (required for Spanish businesses from 2026)
  - Submits simplified (F2) invoices to AEAT; UI fully async, never blocks on the network
  - Submission deferred to payment: tickets saved unpaid stay PENDIENTE and auto-submit when paid at pickup
  - QR + "Factura verificable en la sede electronica de la AEAT" legend printed on ENVIADA tickets
  - Invoice cancellation: Herramientas -> Anular factura Verifactu
  - Invoice rectification (R1-R5, sustitucion / diferencias): Herramientas -> Rectificar factura Verifactu (Art. 8.2.a RD 1007/2023)
  - AEAT XML export by date range: Herramientas -> Exportar registros AEAT (XML) (Art. 14.1 RD 1007/2023)
  - Declaracion responsable del fabricante dialog: Ayuda -> Acerca de Verifactu (Art. 13 RD 1007/2023)
  - AEAT chained hash (Huella, SHA-256) persisted per row for local tamper detection (Art. 12 RD 1007/2023)
  - Verifactu status, CSV, AEAT validation link and "Reintentar envio" accessible per ticket in RecogPrendas
  - "Probar conexion" button in settings validates ServiceKey and endpoint reachability
  - Cancelled (ANULADA) and superseded-by-substitution (RECTIFICADA) rows excluded from quarterly income totals
  - TESTING and PRODUCTION environments supported; v8.0 ships with TESTING (switch after meeting with IreneSolutions)
  - Graceful degradation: unconfigured Verifactu does not block ticket saving
- New feature: centralised settings - all hardcoded paths and values moved to ~/.laideal_settings.json
  - Covers DB path, business identity (name / address / city / phone), IVA rate, single reports root, Verifactu credentials, print toggle
  - Settings dialog accessible via Archivo -> Configuracion; first run or missing DB path opens it automatically
  - Legacy ~/.laideal_cfg and ~/.verifactu_key migrated on first launch
- New feature: persistent debug log at ~/.laideal.log (5 MB rotation, thread-safe)
  - Every DB write logs at debug level; failures log a warning with the SQL error text
  - Archivo -> Log de depuracion shows the path and opens the file directly
- Accounting fixes
  - ANULADA and RECTIFICADA Verifactu rows excluded from quarterly income totals
  - Fixed gastos quarter-boundary double-count (half-open date interval)
  - Contextual confirmation dialogs in contabilidad (lock / revert)
- Receipt and printing improvements
  - Ticket header sourced from AppSettings (business name, NIF, address, city, phone)
  - General conditions rewritten for legal correctness (RD 1453/1987) + RGPD first-layer notice
  - Conditions printed on client copy only; establishment copy omits them
  - Thermal printer column widths (4 / 20 / 7.5) and page margins (0.2 / 0.1 / 0.1 / 0.4) tuned on the physical printer
  - Print toggle in settings replaces the old debug mode (Excel still generated for inspection)
- Search and UX
  - Diacritic-insensitive search in RecogPrendas and Listado ("garcia" matches "Garcia")
  - Ingresos Listado loads most-recent rows first; clickable Verifactu validation URL column
  - Herramientas menu reorganised; Anular and Rectificar actions added
  - Per-garment payment disabled to prevent AEAT duplicate-InvoiceID rejection (pay via "Pagar todo")
- Code quality
  - Coding guidelines applied to all source modules (camelCase identifiers, English comments / debug strings, new-style signal-slot syntax)
  - SQL injection fix in add_garment search (parameterised query)
  - Em-dash and other non-ASCII punctuation purged from sources (Windows code-page hazard)
  - Verifactu module rewritten to fully async API (request ID + requestFinished signal)
- Build: version number managed from root CMakeLists.txt and propagated to the generated version.h
