## Caveats

- `VerifactuManager::getValidationUrl(const VerifactuInvoice&)` is public but used only internally inside `generateQRAsync`. Mirrors `VerifactuConfig::getValidationUrl` so the public placement looks deliberate ŌĆö left as live. Could be demoted to private as a defensible cleanup.
- `VerifactuTaxItem::getDescription()` is read only inside the same class's `toJson()` ŌĆö alive.
- The "newly dead" `VerifactuInvoice` / `VerifactuTaxItem` setters were previously transitively dead via `createAndSubmitInvoice`. Removing them is safe; the matching getters can also go (e.g. `setBuyerNIF` / `getBuyerNIF` go together).
- The Verifactu module surface remains wider than what the F2-simplified retail flow needs. Anything with buyer-NIF / -name or a non-`SIMPLIFIED` `InvoiceType` is unreachable from production paths.
- Removing dead methods is not safe without checking external consumers. This codebase is self-contained (no other repos link against these libraries), so the verifactu cleanup is low-risk if you want to do it.
- This report is a snapshot ŌĆö rerun the agent to refresh it.
